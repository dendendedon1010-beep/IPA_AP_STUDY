# IPA過去問投入ガイド

AP Study v2.3.0では、IPA（情報処理推進機構）が公開する応用情報技術者試験（AP）の過去問を、安全に年度別投入するためのカタログ、配置方針、検証ルールを整備しています。問題本文を追加する前に、利用対象の公式資料、問題番号、正答を必ず原資料で照合してください。

## 取り込み対象

- 応用情報技術者試験の午前問題（問題文、4選択肢、公式正答）
- AP Studyが独自に作成する正答理由、誤答理由、要点、キーワード
- 将来対応として、午後問題の長文、設問、解答例、図表、自己採点情報

`src/data/ipaPastExams.ts` は年度・期・午前/午後と公式資料の所在を管理するカタログです。問題本文はカタログに入れず、`src/data/questions/ipa/ap/` 以下の年度別ファイルへ分離します。URLは公式資料との対応を確認できた場合だけ設定し、未確認ならプロパティ自体を省略します。

## 午前問題のIDルール

形式は `ap-rXX-<season>-am-qNNN` です。年度は令和の2桁、問番号は3桁ゼロ埋めとします。

- `ap-r07-spring-am-q001`
- `ap-r07-spring-am-q080`
- `ap-r06-autumn-am-q001`

`season` は原則 `spring` 又は `autumn` を使います。既存IDは回答履歴、ブックマーク、復習スケジュール、模試履歴に利用されるため変更禁止です。

## 午後問題のIDルールと投入方針

大問単位は `ap-rXX-<season>-pm-qNN`、設問単位まで分ける場合は末尾に `-subNN` を付けます。

- `ap-r07-spring-pm-q01`
- `ap-r07-spring-pm-q02`
- `ap-r06-autumn-pm-q01`
- `ap-r07-spring-pm-q01-sub01`
- `ap-r07-spring-pm-q01-sub02`

午後問題は午前問題の4択型とは別型にします。長文、複数設問、解答例、自己採点を扱える構造とし、図表や表を無理にテキストだけへ変換しません。午後問題本文は **v2.6.0以降で専用ビューアを実装してから** 投入します。v2.3.0ではカタログとID方針だけを管理します。

## 出典表示ルール

### IPA公開資料を正確に引用する問題

- `isQuoteFromIpa: true`
- `sourceName`: `情報処理推進機構（IPA） 応用情報技術者試験 令和7年度 春期 午前 問1` のように年度、期、午前/午後、問番号まで記載
- `sourceUrl`: 当該問題を確認できるIPA公式ページ又は公式PDFのHTTPS URL
- 問題文と選択肢は原資料から改変せず、転記後に再照合
- 解説は公式解説の転載ではなくAP Studyが独自作成

画面では次の意味が明確になるよう表示します。

```text
出典：情報処理推進機構（IPA） 応用情報技術者試験 令和7年度 春期 午前 問1
問題文・選択肢はIPA公開資料より引用
解説はAP Studyが作成
```

### AP Studyオリジナル問題

- `isQuoteFromIpa: false`
- `sourceName: 'AP Study original'`
- IPA公開問題を丸写しせず、応用情報対策用の独自問題として作成
- 解説もAP Studyが作成

```text
出典：AP Study original
問題文・選択肢はAP Studyのオリジナル問題
解説はAP Studyが作成
```

## 問題追加時のチェック項目

1. IPA公式資料と年度、期、午前/午後、問番号が一致している。
2. 問題文、選択肢、公式正答を原資料と再照合した。
3. IDが命名規則に合い、既存ID・旧IDと重複していない。
4. `field` は `src/data/fields.ts` の既定値を使用し、`subField` も空でない。
5. 選択肢はア・イ・ウ・エが各1件で、`correctAnswer` もそのいずれかである。
6. `sourceName` と確認済みの `sourceUrl` が設定されている。
7. 正答理由、誤答理由、要点、キーワードを独自に作成した。
8. 年度別ファイルを `src/data/questions/ipa/ap/index.ts` へ登録した。
9. カタログの取り込み状況と、確認できた公式URLを更新した。
10. 通常演習、検索、分野フィルタ、復習、午前模試で既存問題が利用できることを確認した。

## 検証コマンド

```bash
npm run verify:questions
npm run build
npm run verify:dist
```

`verify:questions` は、ID、重複、4択、正答、分野、解説、IPA引用時の出典、カタログの必須項目とURL形式を検証します。検証を通すために出典を推測したり、チェックを緩めたりしてはいけません。

## 直近5年分を投入する手順

1. IPA公式サイトで対象5年分の年度・期と公開資料を調査する。
2. URLと資料内容を確認し、カタログを午前/午後単位で更新する。
3. 1年度・1期・午前の単位でファイルを作り、原資料から問題を転記する。
4. 10問程度の小さな単位で二重チェックし、解説を独自作成する。
5. 年度別ファイルを集約indexへ登録し、`verify:questions` を実行する。
6. 80問単位の件数、問番号、正答一覧を原資料と照合する。
7. ビルドと画面確認を行い、年度・期単位の小さなPRに分ける。
8. 午後問題はカタログだけ更新し、専用型・ビューア完成前には本文を投入しない。

## 全量投入時の注意点

- `App.tsx` やカタログへ巨大な問題配列を直書きしない。
- カタログと問題本文を分離し、年度・期・午前/午後単位でファイル分割する。
- 将来の動的importを妨げないよう、集約indexから年度別モジュールを参照する。
- 既存IDを変更しない。やむを得ない移行は `legacyIds` とストレージ互換性を確認する。
- IPA引用とオリジナル問題を混在させず、引用フラグと出典を必ず照合する。
- 推測URL、非公式転載、OCRだけで未照合の本文、公式解説の転載を登録しない。
- 図表を欠落させると設問の意味が変わる場合は投入を保留する。
- 一度に大量変更せず、年度・期ごとに検証可能なPRへ分割する。
- 全量化後は初期バンドル容量を測定し、必要に応じて年度別の遅延読み込みを導入する。
